# OpenLens — Agent Context

## What this is
Open source contribution intelligence platform. User pastes a GitHub repo URL, answers 5 repo-specific quiz questions, picks an issue, and gets a full contribution guide — files to touch, commit style, PR template, maintainer patterns — all generated by Claude Haiku from real repo data.

## Local dev commands

```bash
# infrastructure
docker compose up -d

# backend (from /backend)
source ../.env && GITHUB_TOKEN=$GITHUB_TOKEN ANTHROPIC_API_KEY=$ANTHROPIC_API_KEY mvn spring-boot:run

# frontend (from /frontend)
npm run dev   # runs on localhost:5173
```

Backend: localhost:8080
Frontend: localhost:5173 (Vite proxies /api/* to backend)

## Architecture — backend

Hexagonal architecture. Domain layer has zero framework dependencies.

```
api/                    — controllers, DTOs, CORS, exception handler
auth/                   — email/password auth (JWT + bcrypt)
  config/SecurityConfig — Spring Security, stateless, JWT filter
  controller/AuthController — register, login, logout, me
  domain/UserEntity + UserRepository
  filter/JwtAuthFilter  — reads ol_token httpOnly cookie or Bearer header
  service/AuthService   — bcrypt(cost=12), email normalization
  service/JwtService    — HS256, 7-day expiry
  service/LoginRateLimiter — 5 attempts/IP/15min, in-memory
application/service/    — use case orchestration
domain/model/           — Repository, Issue, PullRequest, User, etc.
domain/port/input/      — use case interfaces
domain/port/output/     — repository/external service interfaces
infrastructure/
  ai/                   — Claude Haiku API client + analysis service
  github/               — GitHub REST API client (virtual threads)
  kafka/                — producer + consumer for ingestion jobs
  persistence/          — JPA entities + adapters
  redis/                — cache adapter
```

## API endpoints

```
POST /api/auth/register         — { email, password, name } → JWT cookie
POST /api/auth/login            — { email, password } → JWT cookie
POST /api/auth/logout           — clears cookie
GET  /api/auth/me               — returns current user

POST /api/repos/analyze         — { repoUrl } → triggers Kafka ingestion
GET  /api/repos/status?url=...  — PROCESSING | READY | FAILED

GET  /api/quiz/{repoId}/questions   — 5 AI-generated questions
POST /api/quiz/{repoId}/submit      — answers → skill level + matched issues

GET  /api/guide/{repoId}/issues/{issueId}  — full contribution guide
```

## Kafka topics

`repo.ingestion.requested` — payload: { owner, repoName }

Consumer picks up job → fetches GitHub data → saves issues + PRs + language → marks READY

## DB schema (Flyway)

V1 — repositories (id, url, owner, name, primary_language, stars, status, last_analyzed_at)
V2 — issues (id, repo_id, number, title, body, labels, state, complexity_score)
V3 — pull_requests (id, repo_id, number, title, files_changed, lines_added, lines_removed, merge_time_hours, linked_issue_number, author, merged_at)
V4 — users (id, email, password_hash, name, created_at)
V5 — user_sessions (id, user_id, token_hash, expires_at, created_at)

## AI layer

ClaudeApiClient — Java 21 HttpClient with virtual threads, calls claude-haiku-4-5-20251001
AiAnalysisService:
  generateQuizQuestions() — sends repo language + issue titles + PR titles → 5 JSON questions
  generateContributionGuide() — sends full issue body (2000 chars) + PR patterns → 8-step JSON guide
Falls back to rule-based if Claude API unavailable or returns error.

Requires ANTHROPIC_API_KEY in env. If missing or has no credits, rule-based fallback runs silently.
